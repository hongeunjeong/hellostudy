1. WEB

대인덱스 목록
환경 태그 태그 설명
Web meta name=”subject” content=”meta태그 공부” 홈페이지 주제
meta name="title" content=”meta태그” 제목
meta name="author" content=”eunjeong” 글쓴이
meta name="description" content=”meta태그 정리페이지 입니다.” 설명
meta name="keyword" content=”html, html5, css. css3” 키워드
meta name="generator" content="EditPlus” 저작프로그램
meta name="reply-to" content="janma10101@naver.com” 페이지에 관한 문의처 메일 주소 명시
meta name="content-language" content="kr” 페이지에 기술된 언어 명시
meta name="Robots" content="index, follow” index, follow -> 이 문서도 긁어가고, 링크된 문서도 긁어감
noindex, follow -> 이 문서도 긁어가지말고, 링크된 문서만 긁어감
index, nofollow -> 이 문서는 긁어가고, 링크는 무시
noindex, nofollow -> 이 문서도 긁지않고, 링크도 무시
meta charset="UTF-8" 페이지의 문자 인코딩을 지정.
대표적으로 UTF-8(유니코드를 위한 문자 인코딩 방식 중 하나)를 쓰는게 일반적임.
ASCII와 같은 문자 인코딩은 한글을 표현불가.
meta http-equiv="content-type" content=”text/html; charset=문자 인코딩” http-equiv -> 글을 읽기전 먼저 웹브라우저에 정보를 전달하는 역할
content-type -> 글의 종류로 문자 인코딩 지정을 위해 사용
meta http-equiv="content-style-type" content=”text/css” css사용시 지정
meta http-equiv="pragma" content=”no-cache” 캐쉬가 쌓이지 않도록 해줌.
항상 홈페이지 내용을 새로 읽어들임.
meta http-equiv="refresh" content=”4, http://www.naver.com”

refresh -> 일정 시간이후에 새로고침, 새로고침되기까지 시간을 초 단위로 나타냄.

http://www.naver.com -> 해당 사이트로 이동
meta http-equiv="X-UA-Compatible" content=”IE=Edge, chrome=1” 호환성 강제 렌더링, ActiveX, 기타 스트립트 등 IE 상위버전까지 지원되지 않아 호환성 문제 발생 시 해결
IE=Edge, chrome=1
-> chrome=1을 쓰면 가능한 경우 크롬 프레임 사용
IE=Edge -> IE8이상 버전에서 항상 최신 표준 모드로 렌더링
IE=5 -> IE5 렌더링 방식 사용
IE=7 -> IE 표준모드 렌더링 방식 사용
IE=EmulateIE7 -> IE7 표준모드나 퀵스모드로 렌더링
IE=8 -> IE8 표준모드로 렌더링
IE=EmulateIE8 -> IE8 표준모드나 퀵스모드로 렌더링

2. MOBILE&RESPONSIVE

대인덱스 목록
환경 태그 태그 설명
Mobile meta name="viewport” content=”width=device-width, initial-scale=1, maximum-scale=1.0, minimum-scale=1.0, user-scalable=no, target-densityDpi=medium-dpi” viewport -> 브라우저에서 표시되는 화면영역
webkit 브라우저(크롬, 사파리)의 기본 뷰포트는 980px -> device-width를 맞춰줘야함.
width=device-width -> viewport를 device의 너비만큼 쓰겠다고 선언.

initial-scale -> 초기화면배율
기본값 1 (1보다 크게 설정된 경우 확대된 페이지로 나옴)

maximum-scale -> 최대화면비율
핀치 줌인 시 최대 몇 배율까지 보여줄지 설정

user-scalable -> 사용자의 확대보기를 허용 여부
0 : 허용하지않음 / 1 : 확대보기 허용함

target-densityDpi=medium-dpi
-> 모바일기기에서 동일한 해상도로 보여지도록함 (안드로이드만적용)
